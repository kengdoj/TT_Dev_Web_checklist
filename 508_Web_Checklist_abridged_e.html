Developer's 508 Checklist for Desktop Web Applications

(version 1.e)

Developers are encouraged to become familiar with Section 508 compliance in the early stages of development to ensure applicable requirements are met. Upon completion of the project, developers should complete this self-assessment to assist project managers and Trusted Testers[1] in their Section 508 evaluation. Trusted Tester training registration can be requested by emailing accessibility@hq.dhs.gov.

C. Self-Assessment Checklist

Web developers should review their application and complete the rest of the checklist by selecting the appropriate status for each requirement. Some requirements (like keyboard access) always apply.

508 Category Test ID
(ref)		 Requirements for 508 compliant content Status Explain any applicable requirements that were 'Not Met'
1.1 Keyboard Access A, B All interactive elements and functions can be accessed or activated by keyboard.
C If non-standard or alternative keyboard commands are required for access, they are documented within the application (for example, in the Help section)
D All information revealed by mouse over (TITLE) is available to keyboard-only users (i.e., there is equivalent screen text and/or visual context).
E, F At all times, there is a visual indication of the current keyboard focus and it is on the correct element.
G The visual focus remains within a modal dialog box until closed.
H The visual focus moves to revealed content or a description of the content change is provided.
I The tab order is logical.
1.2.2 Forms A, B, C All Web form fields have markup to associate to the forms complete instructions and cues.
D, E All ARIA forms have complete instructions in the Name property (using aria-labelledby, e.g.), and have correct Role, State, Value properties.[2]
F All form controls identify their purpose.
1.2.3 Links and User Controls A All links have a unique and meaningful description.
B All scripted elements have a unique and meaningful description.
2.2 Images A, B, C, D All meaningful Web images have equivalent text descriptions (purpose and function) via an ALT, TITLE or ARIA attribute.
E A CAPTCHA image ALT describes its purpose.
F All multi-state components provide complete status information.
G Images have consistent meaning throughout the application.
2.3 Video-only and Animation B Video-only files have an equivalent text or audio description.
2.4 Audio-only A Audio files have an equivalent text description/transcript.
2.5 Image Maps A Server side image maps are not used.
3.1 Color Dependence A Information that is provided by color also has onscreen text descriptions or other non-color based differences.
3.2 Color Contrast for Text B All meaningful text and their backgrounds have a minimum contrast ratio of 4.5:1.[3]
4 Flashing A, B All flickering elements have a defined flicker frequency set at or below 3Hz.
5 Page Titles A All pages have a meaningful page title in plain language.
6 Multimedia A, B Equivalent, synchronized captions are provided for the audio content of multimedia.
C, D Equivalent, synchronized audio descriptions are provided for the visual content of multimedia.
7 Time Outs A, B, C A notice is displayed at least 20 seconds before a time out is about to occur, and the user is offered an option to request more time.
9.1 Language A The correct default language for the page is programmatically set.
B A passage (content, image descriptions, form labels etc.) that differs from the default language of the page is programmatically identified correctly.
9.2 Section Headings A, B Visually apparent headings are programmatically identified (with <H>) and match the visual heading structure.
10 Data Tables A All row and column headers of a data table are identified programmatically.
B, D All data tables are created with proper HTML table markup (not images).
C All data cells of complex data tables are programmatically associated to row and column headers.
11 Style Sheet Dependence A The order of the content may have changed but the content remains logical when style sheets are disabled.
B All relevant content or information from a meaningful image are available when style sheets are disabled.
C Content does not overlap or become illegible when style sheets are disabled.
D Programming code or other confusing elements are not revealed on the page when style sheets are disabled.
12 Frames A, B Frames and iframes have a descriptive Title or Name.
13 Repetitive Content and Links A, B, C There is a skip method that moves the keyboard focus past the repetitive content.
D The relative order of repeated components is consistent with other similar pages.
14 Required Plug-ins A In a public site, the link to download a required plug-in is provided.
B The plug-in required to view content is compliant. (Confirm with vendor accessibility documentation.)
15 Alternative (Accessible) Version A If an alternative page is provided, it contains equivalent information as the primary page.
B An alternative page was provided because the primary site cannot be made compliant.
The SAVE function does not work in Internet Explorer.
